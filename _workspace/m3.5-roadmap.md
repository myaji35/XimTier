## 4. 홈페이지 P0 — Reverse What-If 데모 상세

### 4.1 사용자 입장 시나리오
1. 방문자가 `/ko/how-it-works` 또는 홈 페이지 데모 섹션 진입
2. "목표 불량률을 입력하세요" 슬라이더 (0.3% ~ 3.0%, 기본 1.5%)
3. 슬라이더 드래그 즉시 5개 변수 카드 업데이트
   - **온도**: 215°C → optimal °C
   - **압력**: 80 bar → optimal bar
   - **습도**: 45% → optimal %
   - **속도**: 120 rpm → optimal rpm
   - **체류시간**: 3.5s → optimal s
4. 탭 클릭 → SHAP 시각화 (양/음 막대, 중앙 0선 기준)
5. "이런 변수를 조합하면 목표 달성 가능" 자신감 라벨 (R² 0.887)

### 4.2 기술 구현
```javascript
// app/javascript/controllers/reverse_what_if_controller.js
import { Controller } from "@hotwired/stimulus"

export default class extends Controller {
  static targets = ["slider", "value", "variableCard", "shapBar"]
  static values = {
    baseline:  { type: Object, default: { temp: 215, pressure: 80, ... } },
    weights:   { type: Object, default: { temp: 0.34, pressure: -0.21, ... } },
    r2:        { type: Number, default: 0.887 }
  }

  update() {
    const target = parseFloat(this.sliderTarget.value)
    const delta = target - 1.5  // baseline target
    this.variableCardTargets.forEach((card, i) => {
      const key = card.dataset.var
      const optimal = this.baselineValue[key] + delta * this.weightsValue[key] * 10
      card.querySelector("[data-value]").textContent = optimal.toFixed(1)
    })
  }
}
```

### 4.3 산출물
- `app/javascript/controllers/reverse_what_if_controller.js`
- `app/views/shared/_reverse_what_if_demo.html.erb` (ViewComponent로 격상 가능)
- `config/locales/{ko,en}/demo.yml` (변수명/단위/문구)
- RSpec: `spec/system/reverse_what_if_demo_spec.rb` (Capybara + headless Chrome)
