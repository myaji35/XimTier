## 6. 발견 & 학습

### 발견
1. **ISS-003 사전 구현됨** — application.css에 이미 4 chip × float-y 6s가 들어 있어 코드 변경 없이 검증만으로 종료. 사전 명세와 실제 구현 진척의 갭을 매 스프린트 시작 시 자동 스캔할 가치 있음.
2. **ISS-009 (count_up_controller) 회귀 발견** — ISS-003 검증 중 콘솔 에러로 감지. Stimulus base의 `targets` getter-only 속성과 인스턴스 변수명 충돌. 캐릭터 저니 검증이 없었다면 production까지 갔을 케이스.
3. **shap_legend_html 색 토큰 비동기** — ISS-002 색 분리(teal/blue) 적용 시 기존 legend html이 옛 색(blue/red)을 안내했음. 컴포넌트 변경 시 locale 도큐먼트도 동기 점검 필요.

### 학습
- **Stimulus 베이스 getter 충돌**: 향후 컨트롤러에서 `this.targets`, `this.values`, `this.classes` 같은 base API 이름을 인스턴스 변수로 쓰지 않는다. `targetEls`, `valueState` 등 명시적 접미사를 권장.
- **Capybara `aria-valuenow` 정규화**: 컨트롤러가 connect 직후 `toFixed(2)`로 갱신하므로 "1.2" ≠ "1.20". spec은 컨트롤러 출력 형식을 그대로 기대해야 함.
- **`@media (prefers-reduced-motion: reduce)` 검증 패턴**: Playwright 자체는 emulation API 노출이 제한적이라 CSSOM(`document.styleSheets`)을 직접 enumerate해서 룰 존재를 확인하는 방식이 안정적.
- **9차 배포 안정성**: M3 7차 학습(원격 amd64 builder, kamal-proxy + nginx vhost 다중 등록)이 그대로 효과. 7~9차 무사고 연속.
