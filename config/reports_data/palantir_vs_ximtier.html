1. 한 줄 포지셔닝과 시장 정의

Decision Intelligence(이하 DI) 시장은 2030년 기준 TAM $81B, CAGR +28.5%로 추산된다(Gartner·IDC·M&M 기반). 이 시장 안에는 "정부·초대형 기업의 데이터 통합" 영역과 "중견·중소 규제 산업의 일상적 의사결정" 영역이 공존하지만, 두 영역의 고객·구매 절차·예산은 사실상 다른 시장이다.

Palantir의 시장 정의

  • 국방·정보기관·F500의 데이터 통합 OS
  • Ontology 위에서 운영을 시뮬레이션
  • "우리 회사 전체 데이터를 한 운영체제에 올린다"
  • 대형 변혁 프로젝트, CIO·CDO 후원

XimTier의 시장 정의

  • 규제 산업 중견기업의 설명 가능한 의사결정 엔진
  • SHAP·회귀·Reverse What-If로 증명
  • "기존 데이터로 다음 분기 결정을 내린다"
  • 현장 KPI 개선, 공장장·CFO·컴플라이언스 후원
"ChatGPT가 설명한다면, XimTier는 증명한다. Palantir가 데이터를 통합한다면, XimTier는 통합된 데이터 위에서 결정의 수학을 푼다."

3-1. 왜 LLM/AIP로는 "증명"이 안 되는가

Transformer 아키텍처는 다음 토큰을 확률적으로 예측하는 모델이다. 회귀 계수·SHAP 값·Reverse What-If 최적화를 수학적으로 보장하지 않는다. "그럴듯한 숫자"는 출력하지만 EU AI Act 9조(고위험 AI의 수학적 근거 제시 의무)·한국 AI 기본법 13조(설명 가능성)에 맞춰 감사를 통과시키지 못한다. Palantir AIP도 LLM 위에 얹은 레이어이므로 동일한 한계를 상속받는다.

LLM (언어 인터페이스)

  • ✓ 일반 지식 검색 · 텍스트 생성
  • ✗ 실제 통계 계산 (회귀·SHAP)
  • ✗ Reverse What-If — 구조적 불가
  • ✗ 동일 입력 → 동일 출력 (확률적)
  • ✗ 규제 XAI 의무 충족

XimTier (계산 엔진)

  • ✓ Reverse What-If — 목표 → 변수 역산
  • ✓ SHAP 기반 수학적 XAI
  • ✓ 온프레미스 / 프라이빗 — 데이터 외부 유출 없음
  • ✓ 동일 입력 → 동일 출력 재현성
  • ✓ 규제 통과 가능 보고서

6. 해자(Moat) — Palantir·빅테크가 들어올 수 없는 교차점

XimTier의 해자는 세 교차점에서만 만들어진다. 이 교차점은 Palantir의 BM 구조와 빅테크 LLM의 아키텍처적 한계 때문에 의도적으로 비워진 시장이다.

① 데이터 주권 Strategic

클라우드 사업이 본업인 Palantir Apollo·OpenAI·MS는 온프레미스 100% 모델로 내려올 수 없다. 자기 BM 자기잠식이기 때문이다.

② 규제 통과 수학 Architectural

LLM 기반 빅테크 제품은 "수학적 증명"이 구조적으로 불가능. Transformer는 다음 토큰 예측기이지, 회귀·역최적화 풀이기가 아니다.

③ Reverse What-If Algorithmic

"목표 → 변수 역산"은 LLM 추론으로 풀 수 없는 최적화 문제. 회귀·SHAP·제약조건 기반 별도 엔진이 필수다.

④ 한국·아시아 현지성 Geographic

Palantir 본사 미국, 한국 인력 풀 제한, 한글 UI/도메인 케이스 부족. XimTier는 처음부터 한국 규제·언어·산업 케이스에 맞춰 설계.

해자 요약 — Palantir가 우리 시장에 내려오려면 자기 클라우드 BM을 깨야 하고, 자기 AIP의 LLM 의존을 부정해야 한다. 그 가능성은 매우 낮다. 빅테크 본격 대응 전 선점 윈도우는 18~24개월.

8. 예상 반론 및 답변 (FAQ)

Q1. Palantir가 SMB 버전을 출시하면 XimTier는 끝나는 것 아닌가?

Palantir BM의 절반은 FDE(Forward Deployed Engineer) 인건비다. SMB 가격대(연 ₩1억 이하)로 내려오면 단위 경제가 깨진다. 시도하더라도 한국·아시아 규제 산업에 우리가 2~3년 먼저 도달해 있을 것이고, 그 시점엔 케이스 50+로 진입장벽을 만든 상태다.

Q2. OpenAI가 온프레미스를 시작했는데 차별점은?

OpenAI의 온프레미스는 LLM을 사내에 두는 것이지 수학적 근거를 만드는 것이 아니다. EU AI Act는 LLM으로는 통과 불가. XimTier는 OpenAI 온프레미스 옆에 나란히 설치되는 보완재다. "ChatGPT는 발표, XimTier는 증명".

Q3. 결국 BI 도구(Tableau·Power BI)와 뭐가 다른가?

BI는 대시보드를 만든다. XimTier는 의사결정을 만든다. BI 대시보드는 "지난 분기 불량률이 2.4%였습니다"를 보여주지만, XimTier는 "불량률을 1.2%로 낮추려면 온도를 215°C → 207°C로 조정"이라는 역산 결정을 SHAP 근거와 함께 출력한다.

Q4. SAS·SPSS 같은 전통 통계 도구는?

전통 통계는 분석가 전용. XimTier는 비전문가가 직접 5분 안에 사용하도록 설계됐다. 구축 비용 1/10, 도메인 즉시 적용, 바이브 코딩 연계로 커스텀 빠르게.

Q5. DataRobot·Databricks는?

DataRobot은 AutoML, Databricks는 데이터 레이크. 둘 다 플랫폼이며 데이터 엔지니어가 필요하다. XimTier는 의사결정 결과물 자체를 보장하며, 도메인 패키지로 즉시 사용된다.

Q6. Palantir와 파트너십이 가능한가?

가능하다. Palantir가 못 가는 한국 중견 시장에 우리가 먼저 들어가고, 우리 케이스가 쌓이면 Foundry 데이터 커넥터 형태로 흡수가 아닌 공존이 열린다. 시나리오 B/D가 그 경로다.
